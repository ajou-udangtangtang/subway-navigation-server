# 07. 구현 계획

## 7.1 기술 스택

### 7.1.1 서버 (백엔드)

| 분류 | 기술 | 사유 |
|---|---|---|
| 언어 | Python 3.10+ | 데이터 처리·KNN 구현이 용이함, 팀 친숙도 |
| 웹 프레임워크 | Flask | 경량 프레임워크. 학습 부담이 적고 본 시스템의 단순한 API 구조에 적합 |
| DB 드라이버 | PyMySQL | Python ↔ MySQL 표준 드라이버 |
| 데이터베이스 | MySQL 8.x | 팀 친숙도, fingerprint 데이터 보관 |
| 위치 추정 | scikit-learn (KNN) 또는 직접 구현 | 단순 거리 기반이라 직접 구현도 어렵지 않음 |

### 7.1.2 클라이언트 (앱 측, 참고)

| 분류 | 기술 | 비고 |
|---|---|---|
| 플랫폼 | Android (네이티브) | Galaxy S24 대상 |
| 통신 | Retrofit | Android 표준 HTTP 클라이언트 |
| 센서 | SensorManager API | 자기센서(나침반) 데이터 |
| Wi-Fi | WiFi Scan API | 주변 AP 스캔 |
| 진동 | Vibrator API | 진동 출력 |
| 보조기능 | TalkBack | 시스템 보조기능, 자동 호환 |

## 7.2 서버 폴더 구조

### 7.2.1 디렉토리 트리

```
subway-navigation-server/
├── app.py                  # Flask 진입점, 라우팅 정의
├── config.py               # 환경 설정 (DB 접속 정보 등)
│
├── api/                    # API 핸들러 (선택적 분리)
│   ├── __init__.py
│   ├── locate.py           # /locate 엔드포인트 로직
│   ├── route.py            # /route 엔드포인트 로직
│   └── direction.py        # /direction 엔드포인트 로직
│
├── core/                   # 도메인 로직
│   ├── __init__.py
│   ├── graph.py            # 그래프 로드 + Dijkstra
│   ├── direction.py        # atan2 기반 절대 각도 계산
│   ├── knn.py              # KNN (팀원 B 담당)
│   └── wifi.py             # AP 정렬, 결측 -100 채움
│
├── db/                     # DB 연동
│   ├── __init__.py
│   ├── connection.py       # PyMySQL 커넥션 풀
│   └── fingerprint.py      # fingerprint 테이블 조회
│
├── data/                   # 정적 데이터
│   ├── nodes.json
│   ├── edges.json
│   └── danger.json
│
├── tests/                  # 테스트 (시연 시나리오 검증용)
│   ├── test_graph.py
│   └── test_direction.py
│
├── requirements.txt
├── README.md
└── docs/                   # 본 문서 디렉토리
    └── ...
```

### 7.2.2 모듈 책임 요약

| 모듈 | 책임 | 담당 |
|---|---|---|
| `app.py` | Flask 앱 생성, blueprint 등록, 서버 기동 | A |
| `api/*.py` | HTTP 요청 검증, JSON 직렬화, 응답 형식 통일 | A |
| `core/graph.py` | 그래프 JSON 로드, Dijkstra (위험 노드 제외) | A |
| `core/direction.py` | 두 노드 좌표 → 절대 각도 산출 | A |
| `core/wifi.py` | AP 정렬, -100 채움 (KNN 호출 직전) | A |
| `core/knn.py` | KNN 알고리즘 본체 | **B** |
| `db/*.py` | PyMySQL 연결, fingerprint 조회 | B |
| `data/*.json` | 그래프 데이터 (수동 작성) | A 작성, 공동 검증 |

### 7.2.3 최소 구성 (MVP) 옵션

학습 부담을 줄이기 위해, 초기 구현에서는 다음과 같이 더 단순한 구조로 시작할 수 있다.

```
subway-navigation-server/
├── app.py          # 모든 API 엔드포인트
├── graph.py        # Dijkstra
├── direction.py    # atan2
├── knn.py          # KNN (B)
├── db.py           # PyMySQL 연결
├── data/
│   ├── nodes.json
│   ├── edges.json
│   └── danger.json
└── requirements.txt
```

> 동작 검증이 우선이므로 **MVP 구조로 시작**하고, 코드가 안정화되면 §7.2.1의 정식 구조로 리팩토링할 것을 권장한다.

## 7.3 단순화 원칙 (성능보다 완성)

### 7.3.1 본 프로젝트의 우선순위

본 프로젝트는 **"실제로 동작하는 어플리케이션"의 완성**을 최우선 목표로 삼는다. 이를 달성하기 위해 다음과 같은 의사결정 원칙을 채택한다.

> "동작하지 않는 우아한 설계보다, 거칠더라도 동작하는 구현이 우선한다."

### 7.3.2 단순화 대상 항목

| 원본 요구사항 | 단순화 적용 | 사유 |
|---|---|---|
| "3초간 측정 후 다수결" | **1회 측정으로 시작** | 다수결 로직은 동작 검증 후 추가 |
| "AP 정렬 정교하게" | **BSSID 알파벳순 정렬**만 적용 | 일관성만 유지하면 KNN 결과는 동일 |
| "Dijkstra 성능 최적화" | `heapq` 표준 라이브러리 그대로 | 노드 수가 수십 개 수준 |
| "위험 노드 별도 테이블" | **JSON 파일 1개** | 변경이 드물고 단순 조회 |
| "DB에 그래프 저장" | **JSON 파일 1회 로드** | 트랜잭션 불필요 |
| 사용자 인증/세션 | **구현하지 않음** | 요구사항 명세서에도 없음 |
| 통신 재시도/백오프 | **구현하지 않음** | 시연 시나리오에서 발생 빈도 낮음 |

### 7.3.3 단순화하지 않을 항목

다음 항목은 단순화 시 시연 자체가 실패할 수 있으므로 반드시 구현한다.

- **위험 노드 회피 로직**: 시연 시나리오 7번 단계의 핵심
- **AP 정렬 일관성**: 정렬이 깨지면 KNN이 아예 동작하지 않음
- **잘못된 방향 감지**: 시각장애인 안전 관련

### 7.3.4 흔히 빠지는 함정

다음과 같은 "있어 보이는" 작업은 **하지 않는다**.

- 처음부터 Docker로 컨테이너화
- gunicorn/nginx 등 프로덕션 서빙 구성
- 로그 시스템 정밀 설계 (일단 `print`로 충분)
- Swagger 등 자동 API 문서화 도구 도입
- TypeORM/SQLAlchemy 등 ORM 도입
- 단위 테스트 100% 커버리지 달성

위 작업들은 동작이 검증된 이후에 시간이 남으면 도입한다.

## 7.4 외부 라이브러리 의존성

### 7.4.1 `requirements.txt` 초안

```
Flask>=3.0
PyMySQL>=1.1
numpy>=1.26          # KNN 거리 계산용
scikit-learn>=1.4    # (선택) KNeighborsClassifier 사용 시
```

### 7.4.2 라이브러리 선택 사유

| 라이브러리 | 사유 |
|---|---|
| Flask | 경량 웹 프레임워크. `route` 데코레이터만 알면 즉시 시작 가능 |
| PyMySQL | 순수 Python 구현으로 설치가 단순. 별도 컴파일러 불필요 |
| numpy | KNN의 벡터 거리 계산에 유용. 직접 구현 시에도 권장 |
| scikit-learn | (선택) `KNeighborsClassifier` 를 사용하면 KNN 직접 구현 부담을 줄일 수 있음 |

### 7.4.3 개발 환경 가이드

| 항목 | 권장 값 |
|---|---|
| Python 버전 | 3.10 이상 |
| 가상환경 | `venv` 또는 `pyenv` |
| OS | macOS / Linux / Windows 모두 가능 |
| MySQL 설치 방법 | macOS: `brew install mysql` / Windows: 공식 인스톨러 |

### 7.4.4 환경 변수 (예시)

```bash
# .env (또는 config.py 직접 정의)
DB_HOST=localhost
DB_PORT=3306
DB_USER=subway_user
DB_PASSWORD=...
DB_NAME=subway
KNN_K=5
```

> 비밀 값은 `.gitignore` 처리하며, 공개 repo에 올리지 않는다.
